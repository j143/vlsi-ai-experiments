# Datasets Directory

This directory stores CSV/Parquet files generated by design space sweeps and experiments.

## File Naming Convention

- **Analytical reference**: `bandgap_sweep_real_sky130.csv` — pre-generated multi-corner
  reference dataset (analytical Brokaw model, tt/ff/ss corners). Committed to the repo;
  used as the default input for the full pipeline.
- **Real SPICE sweeps**: `bandgap_sweep_<YYYYMMDD_HHMMSS>.csv` — from `data_gen/sweep_bandgap.py`
- **Variant sweeps**: `bandgap_<variation>_<timestamp>.csv` (e.g., corner, temperature)

---

## File Format

Each CSV contains columns:

| Column | Type | Description |
|--------|------|-------------|
| N | int | BJT emitter area ratio |
| R1 | float | Series resistor [Ω] |
| R2 | float | PTAT resistor [Ω] |
| W_P | float | PMOS width [m] |
| L_P | float | PMOS length [m] |
| vref_V | float | Output reference voltage [V] |
| tc_ppm_C | float | Temperature coefficient [ppm/°C] (one-sided, lower is better) |
| psrr_dB | float | Power supply rejection ratio [dB] |
| iq_uA | float | Quiescent current [µA] |
| corner | str | Process corner: "tt", "ff", or "ss" |
| err_from_target_mV | float | \|Vref - 1.2V\| [mV] |
| spec_vref_pass | bool | True if within ±10 mV of 1.2 V target |
| spec_tc_pass | bool | True if TC ≤ 20 ppm/°C (direction: minimize) |
| error | str | Error message if simulation failed (empty if OK) |
| source | str | "analytical" or "ngspice" |

## Injecting Your Own Real Data

To replace the analytical dataset with real SPICE results:

```bash
# Your real sim results — must have the columns listed above
# (at minimum: N, R1, R2, W_P, L_P, vref_V, error)
cp your_real_data.csv datasets/bandgap_sweep_real_sky130.csv

# Then run the full pipeline:
python examples/run_full_pipeline.py --dataset datasets/bandgap_sweep_real_sky130.csv
```

Or regenerate the analytical dataset with more samples:
```bash
python examples/generate_reference_dataset.py --n-samples 500 --save-to-datasets
```

## Data Provenance

When committing a dataset:
1. **Do not commit raw ngspice `.raw` files** (too large, tool-dependent).
2. **Commit only CSV/Parquet summaries** (< 5 MB per file).
3. **Add metadata** in a comment row or separate `<dataset>_metadata.json`:
   ```json
   {
     "timestamp": "2026-03-04T17:00:00Z",
     "pdk": "sky130_fd_pr (TT corner, bundled)",
     "commit": "a1b2c3d",
     "ngspice_version": "ngspice-41",
     "n_samples": 50,
     "n_failed": 0,
     "min_vref_V": 0.71,
     "max_vref_V": 1.15,
     "spec_pass_rate": 0.08
   }
   ```

## Using Datasets in ML

### Load a dataset:
```python
import pandas as pd

df = pd.read_csv("datasets/bandgap_sweep_real_sky130.csv")

# Filter valid runs (drop simulation failures, keep all corners)
valid = (df["error"] == "") | df["error"].isna()
X = df.loc[valid, ["N", "R1", "R2", "W_P", "L_P"]].values
y = df.loc[valid, "vref_V"].values

print(f"Valid points: {X.shape[0]}/{len(df)}")
print(f"Corners: {df['corner'].value_counts().to_dict()}")
```

### Train a surrogate:
```python
from ml.surrogate import GaussianProcessSurrogate

model = GaussianProcessSurrogate()
model.fit(X[:40], y[:40])  # Train on first 40
pred, std = model.predict_with_uncertainty(X[40:])  # Predict on rest
```

### Benchmark:
```python
from sklearn.metrics import r2_score, mean_squared_error

r2 = r2_score(y[40:], pred)
rmse = np.sqrt(mean_squared_error(y[40:], pred))
print(f"R² = {r2:.3f},  RMSE = {rmse:.6f} V")
```

## Size Limits

| File Type | Max Size | Reason |
|-----------|----------|--------|
| CSV (~20 columns × 1000 rows) | 5 MB | Keep diffs manageable |
| Parquet (compressed) | 2 MB | Binary format, reduces size |
| ngspice `.raw` (binary) | Not committed | Tool-specific, use CSV instead |

If a sweep produces > 5 MB, split it into multiple corner/batch files or document a download link in a `_LARGE_FILES.md`.

---

## Workflow

1. **Generate a sweep**: `python data_gen/sweep_bandgap.py ...`
2. **Inspect the CSV**: `python -c "import pandas as pd; df=pd.read_csv(...); print(df.describe())"`
3. **Check for errors**: `df[df['error'] != ''].head(10)`
4. **Save meaningful datasets** to git if they represent a milestone.
5. **Archive old data** if the directory grows beyond 100 MB total.
